    • Group Project, System Analysis and Design

    (January 2022 – May 2022)


       In our System Analysis and Design group project, we followed the System Development Life Cycle (SDLC) framework and stopped just before the coding phase. Collaborating with a small agile team of four other students, our objective was to simulate the SDLC process while employing agile methodology.

    1. Planning
    Objective: Define the scope and purpose of the project.
    Activities: We outlined the project's goals, identified stakeholders, gathered requirements, assigned roles and responsibilities, and created a timeline to guide the process.
    Outcome: This resulted in a clear project roadmap, including timelines and a feasibility study that set the foundation for our project.

    2. Requirements Analysis
    Objective: Understand the business needs and define software requirements.
    Activities: Using object-oriented analysis and design (OOAD) techniques, we engaged with stakeholders to document both functional and non-functional requirements. We created key diagrams, such as use case diagrams to map user interactions and class diagrams to represent object structure.
    Outcome: This effort culminated in a detailed Requirements Specification Document (RSD) that captured our findings.

    3. Design
    Objective: Create a blueprint for the system architecture and user interface.
    Activities: Our team produced a comprehensive design document that included the database schema, high-level and detailed software architecture, user interface wireframes, and considerations for security. We utilized activity and sequence diagrams to visualize system workflows.
    Outcome: The design documents prepared us for the implementation (coding) phase, even though we did not proceed to that stage.

    Results:
    Overall, I found this group project immensely helpful in gaining practical experience with the SDLC and preparing a project for implementation. Each phase we followed allowed us to systematically develop a clear and actionable plan for the system. This experience provided valuable insights into real-world systems analysis and design principles, enhancing my skills in project management and collaboration within a team setting.


    KEY POINTS:
  • Understanding of the SDLC Framework: Gained a comprehensive understanding of the System Development Life Cycle (SDLC) phases and their importance in developing a structured approach to software projects.

  • Application of OOAD Techniques: Utilized object-oriented analysis and design (OOAD) methodologies to effectively gather and document system requirements, leading to the creation of essential diagrams such as use case, class, activity, and sequence diagrams.

  • Collaborative Team Dynamics: Developed essential skills in collaboration and communication within an agile team, learning how to effectively assign roles, delegate tasks, and engage with stakeholders to ensure project alignment.

  • Preparation for Implementation: Produced detailed design documents that outlined system architecture, user interfaces, and security considerations, providing a solid foundation for moving towards the implementation phase of the project.
